# Agent Instructions

This project uses **sd** (seeds) for issue tracking. Run `sd prime` to get started.

## Quick Reference

```bash
sd ready              # Find available work
sd show <id>          # View issue details
sd update <id> --status in_progress  # Claim work
sd close <id>         # Complete work
sd sync               # Sync with git
```

## Trellis Domain

Trellis is the **repo-local planning and workflow artifact tool** for the `os-eco` stack.

- **Specs:** Durable intent, constraints, and references.
- **Plans:** Execution shape and status transitions.
- **Handoffs:** Append-only logs of context/ownership transfer between agents/humans.
- **Events:** Audit trail of all transitions.

## Agent Guidelines

- **Storage:** All state must stay inside `.trellis/` as YAML or JSONL.
- **Git-native:** Artifacts must be diffable and mergeable.
- **Concurrency:** Always use `tl` or the internal `src/system/lock.ts` when writing to ensure multi-agent safety.
- **Strict Mode:** Adhere to the strict TypeScript conventions in `CLAUDE.md`.
- **Positioning:** Trellis is NOT an issue tracker (use `seeds`) and NOT an orchestrator (use `overstory`). It is the DURABLE STORAGE for the planning artifacts generated by humans and orchestrators.
